第三章 基本概念

3.4数据类型

  • 3.4.1 typeof操作符

    • ‘undefined’ 这个值未定义
    • ‘boolean’ 这个值是布尔类型
    • ‘number’ 这个值是数字类型
    • ‘string’ 这个值是字符串类型
    • ‘object’ 这个值是对象或者null
    • ‘function’ 函数

  • undefined类型

    • 变量声明后默认为undefined类型

  • Null类型

    • 空对象的指针
    • 如果一个值准备在将来保存对象，则最好将该变量初始化为null

  • Boolean类型

    • 0，NaN，’’(空字符串)，false，undefined，null为false

  • Number类型

    • NaN：非数值（Not a Number)，特殊的数值

      • 与任何数值都不相等，包括本身 NaN == NaN ：false
      • isNaN(任何类型参数)：确定是否“不是数值”

    • 数值转换

    • Number()

    • parseInt()

  • String类型

    • toString()：undefined、null 没有这个方法
    • String()：可以将任何类型转换为string；
      String(null) => ‘null’
      String(undefined) => ‘undefined’

  • Object类型，每个对象都有的属性和方法

  • 逻辑非（！）

    如果第一个操作数能决定结果，那么就不会再对第二个操作数求值

  • 逻辑非（||）

  • 全等和不全等

  • 条件操作符 ? :

3.7 函数

  • 在函数体内可以通过argumetns来访问所有传入的参数，可通过访问arguments对象的length来获知传入了多少个参数
  • 没有重载，后定义的函数会覆盖新定义的函数，可通过检查传入函数参数类型和数量作出不同反应，可以模仿方法重载

小结

  • es5基础数据类型包括：Undefined、Null、Number、String、Boolean
  • es5中的引用数据类型：Object类型
  • 无指定返回值的函数返回的是undefined
  • es5中没有函数签名的概念，因为函数参数是以一个包含零个或多个值的数组形式传递的

第4章

变量、作用域和内存问题

4.1 基本类型和引用类型的值

  • 复制对象时，这个值的副本实际上是一个指针，这个指针指向存储在堆中的一个对象
  • 检测引用类型instanceof

执行环境及作用域

  • 执行环境定义了变量或函数有权访问的其他数据，决定了它们各自的行为
  • 当代码在一个环境中执行时，会创建变量对象的一个作用域链。作用域链的用途，是保证对执行环境有权访问的所有变量和函数有序访问
  • 没有块级作用域
  • 查询标识符，向上逐级查询与给定名字匹配的标识符，如果找到了就停止搜索
  • 垃圾收集：找出那些不再继续使用的变量，然后释放其占用的内存
  • 管理内存，确保占用最少的内存可以让页面获得更好的性能。优化内存占用的最佳方式，就是为执行中的代码只保存必要的数据，一旦数据不再有用，最好将其设为null来释放其引用（解除引用）。

小结：

1、变量可以保存基本数据类型和引用类型
2、基本数据类型在内存中占据固定大小，保存在栈内存中
3、引用类型的值是对象，保存在堆内存中
4、一个变量向另一个变量复制引用类型的值，复制的是指针，两个变量指向同一个对象
5、typeof：确定基本类型 instanceof：确定引用类型

第6章 面向对象的程序设计

数据属性和访问器属性

构造函数模式

  • 任何函数只要通过new操作符来调用，那它就可以作为构造函数；而任何函数，如果不通过new 那它和普通函数没什么区别
  • 每个函数都有protorype（原型）属性，这个属性是一个指针，指向函数原型的对象，这个对象的用途是包含可以由特定类型的所有实例共享的属性和方法
  • isPrototypeOf()方法来确定对象之间是否存在关系
  • Object.getPrototypeOf() 取得一个对象的原型
  • 可以通过对象实例访问保存在原型中的值，但却不能通过对象实例重写原型中的值，如果在实例中创建同名属性，则会先访问实例
  • hasOwnProperty()方法 可以检测一个属性是否存在实例中还是存在于原型中，在实例中返回true
